# AVL и красно-чёрное дерево на Python

Реализация двух сбалансированных бинарных деревьев поиска на языке Python:
- **АВЛ-дерево** (AVL Tree) – строгая балансировка по высоте;
- **Красно-чёрное дерево** (Red-Black Tree) – балансировка с помощью цвета узлов.

Включены операции **вставки** и **удаления** элементов с автоматической балансировкой, а также обход дерева для проверки упорядоченности.

## 🚀 Возможности

- Вставка нового значения (`insert`)
- Удаление существующего значения (`delete`)
- Поиск элемента (`search` – только для RB, для AVL легко добавить)
- Симметричный обход (`inorder`) для получения отсортированного списка
- Автоматическая балансировка, сохраняющая свойства деревьев

## ⚙️ Требования

- Python 3.6 или выше
- Стандартная библиотека (дополнительные пакеты не требуются)

## 🧪 Пример использования

### АВЛ-дерево

```python
from avl_tree import AVLTree

avl = AVLTree()
for value in [10, 20, 30, 40, 50, 25]:
    avl.insert(value)

print(avl.inorder())        # [10, 20, 25, 30, 40, 50]

avl.delete(30)
print(avl.inorder())        # [10, 20, 25, 40, 50]

📊 Сравнение деревьев
Характеристика	АВЛ-дерево	Красно-чёрное дерево
Балансировка	Жёсткая (разница высот ≤ 1)	Мягкая (высота ≤ 2·log₂(n+1))
Поиск	Быстрее	Чуть медленнее
Вставка / удаление	Медленнее (больше поворотов)	Быстрее (меньше перестроений)
Дополнительная память	Целочисленная высота	Бит цвета
Сложность в реализации	Средняя	Высокая (много случаев при фиксации)
📈 Выводы по работе
Обе структуры обеспечивают логарифмическую сложность O(log n) для поиска, вставки и удаления.

АВЛ-деревья предпочтительны для сценариев с преобладанием операций чтения (например, словари с редкими обновлениями).

Красно-чёрные деревья лучше подходят для динамических данных, где частоты вставок и удалений сравнимы с поиском (стандартная реализация std::map в C++).

Реализация красно-чёрного дерева сложнее из-за необходимости поддерживать цветовые свойства, но итоговая производительность на реальных данных часто оказывается выше при интенсивной модификации.
